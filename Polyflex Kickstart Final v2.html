Project Overview & Objectives

Why we are here, what we are delivering, and who is involved at the principal level.

G1

Digitise 9 machines

Retrofit Printing, Slitting, Dry Lam, SF Lam, and 5 Cutting machines with sensor-grade data capture — 163 points in total (incl. Varos pallet-truck weight).

G2

Unify MES ↔ SQL

Three bidirectional API bridges linking PANDORA MES/ERP with Polyflex's existing SQL Manufacturing Accounting.

G3

Printing-line automation

AOI defect detection, CAIVC viscosity control, and TSSSFE fire suppression across all 9 printing decks.

G4

Compliance & handover

BOMBA certification, FAT/UAT sign-off, train-the-trainer, and 1-year 9×6 remote support in place.

Service Provider

HEXA IoT Sdn. Bhd.

Malaysian industrial IoT and smart-manufacturing solutions. Delivery lead, chair of this kickoff.

Client

Polyflex Packaging Sdn. Bhd.

Flexible packaging manufacturer — plastic bags, pouches, wraps. Site in Batu Gajah, Perak.

Funding Body · SIRIM Smart Tech Up

1 unit

1 unit

Industrial Label Printer

For generating unique QR codes for inventory and batch tracking.

1 unit
AOI

Automated Optical Inspection System

High-speed industrial vision system for real-time print defect detection.
  • Industrial line-scan cameras + LED illumination, up to 300 m/min
  • Auto-classifies missing print, dirty spots, scratches, knife wire, overprint deviation, wrinkles, stains, trailing text, foreign matter
  • Acousto-optic alarm on critical defects; adjustable sensitivity per substrate
  • Inspection logs linked to PANDORA MES for quality traceability
  • Includes installation, cabling, calibration & QC team training
CAIVC

Centralised Automatic Ink Viscosity Controller

Closed-loop ink viscosity control across all 9 printing decks.
  • Non-contact ultrasonic vibration sensors with Teflon-coated, anti-clog surfaces
  • Control unit auto-doses solvent within tolerance for stable colour & minimal waste
  • 10" central touchscreen HMI — recipes, live status, alarms with guidance
  • Pneumatic pumps per deck for continuous anti-settling circulation
  • 304 stainless build, surge protection, e-stop, low-voltage detection — DOSH-compliant
  • Full data link to PANDORA MES & AOI for production tracking
TSSSFE

Temperature-Sensing Self-Starting Fire Extinguisher

Localised, self-activating CO₂ suppression on each printing deck.
  • Temperature-sensing detection tubes auto-trigger at preset thresholds — source-localised, no impact on adjacent decks
  • Independent modular units on all 9 decks; 42 kg CO₂ cylinders + electromagnetic drives
  • Non-conductive, non-corrosive CO₂ — safe for printing components, leaves no residue
  • 304 stainless brackets, release lines, solvent-resistant flexible tubing
  • Spray hoods, safety valves, weighing devices — ISO 9001 + DOSH
  • BOMBA liaison & certification included on award

⚠ Explicitly OUT of HEXA IoT scope — Polyflex responsibility

  • SQL API Development — the development, exposure, and licensing of the SQL API endpoints are not included. Must be provided by Polyflex or its SQL vendor to HEXA IoT for integration purposes.
  • Civil & Infrastructural Works — any structural renovations, flooring reinforcement, or partition builds required for the new systems are excluded.
  • Electrical Supply Upgrades — provision of additional 3-phase power isolators or main switchboard upgrades required to power the new systems or micro server is the responsibility of Polyflex.
  • SQL Manufacturing Accounting Licensing — any costs related to upgrading Polyflex's existing SQL licenses or purchasing additional user seats from the SQL principal are excluded.
  • Network Infrastructure — while HEXA IoT provides the Micro Server, the facility-wide Wi-Fi coverage and LAN cabling to the machine areas must be provided by Polyflex.

Risk Assessment

From the submitted SIRIM risk assessment · 6 risks, all scored Low. Click any row for mitigation detail.

ID
Risk
Category
Responsibility
R1
▶ ERP IntegrationPandora MES ↔ SQL sync failure
Operational
Software Architect
Risk & Impact

Data Integrity Failure — Critical operational failure where Production & Inventory data do not match, halting invoicing capabilities. Likelihood: rarely occur. Impact: quarterly to annually.

Existing safeguard + mitigation action

Secure API & Micro Server in place. Action: Implement a "Store-and-Forward" buffer on the Micro Server to hold data if it disconnects from the internet — ensuring nothing is lost.

R2
▶ Traceability Complianceoperators bypass QR scanning
Intangible
Ops Manager — Polyflex
Risk & Impact

Non-compliance to traceability workflow — operators bypass scanning to speed up work, causing end-to-end traceability failure. Source of raw materials in finished goods becomes unverifiable, preventing targeted recalls during quality incidents and damaging credibility with customers.

Existing safeguard + mitigation action

Rugged tablets & label printers issued. Action: Train-the-trainer approach — the PIC performs regular operator refresher training. Owner sits with Polyflex Ops as the floor authority.

R3
▶ System Stabilityreal-time dashboarding overload
Operational
Project Manager
Risk & Impact

Dashboard freezing / slow performance — streaming data from machines simultaneously overloads the local network. Dashboard lags or freezes; PICs stop trusting the "Real-Time" status and stop using the tool. Likelihood: rarely occur. Impact: once in >1 year or none.

Existing safeguard + mitigation action

Local Micro Server database. Action: Obtain an Micro Server with higher CPU and RAM specs (i7-12th Gen, 64 GB RAM, 1 TB SSD per spec).

R4
▶ Connectivitywireless data transmission interference
Operational
Project Manager
Risk & Impact

Signal packet loss — factory environment has heavy metal machines and motors. IoT sensors disconnect intermittently due to interference, causing some data not to be recorded and leading to missing data in production reports. Likelihood: seldom occur. Impact: once in >1 year or none.

Existing safeguard + mitigation action

Strong WiFi strength within the designated area. Action: Conduct a pre-deployment "RF Site Survey" to identify dead zones before commissioning.

R5
▶ System Maintainability3rd-party SQL API breakage
Strategic
Project Manager
Risk & Impact

Integration failure after SQL software update — relying on SQL's Inventory & Production Order APIs. If SQL updates their API version (e.g. annual update), the connection to Pandora might break unexpectedly, stopping all data sync until fixed. Likelihood: rarely occur. Impact: quarterly to annually.

Existing safeguard + mitigation action

Annual maintenance support contract. Action: Include a clause in the SLA requiring the client to notify HEXA IoT 30 days before any major SQL upgrade.

R6
▶ Device Availabilitytablet & scale battery depletion
Operational
Ops Manager — Polyflex
Risk & Impact

Device power failure during shift — operators forget to charge the tablets or scales (Varos). Devices run out of power mid-shift, forcing operators back to paper/manual recording — breaking the digital process. Likelihood: seldom occur. Impact: once in >1 year or none.

Existing safeguard + mitigation action

Check battery-level indicators on the devices. Action: Supply charging stations near the production line and mandate an SOP — devices must be docked and charged at the end of the shift.
